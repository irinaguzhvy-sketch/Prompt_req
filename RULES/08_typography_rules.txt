# TYPOGRAPHY SYSTEM - STRICT HIERARCHY

## GLOBAL FONT STACK (ALL TEXT)
- Primary: Arial
- Fallback: Helvetica
- Generic: sans-serif
- Implementation: `font-family: Arial, Helvetica, sans-serif;`
- NO other font families allowed
- NO external fonts (Google Fonts, Typekit, etc.)

## TEXT ALIGNMENT RULES
- Main body paragraphs: LEFT-ALIGNED
- Headlines: CENTERED (may be left-aligned in editorial patterns)
- Text directly under CTA button: CENTERED
- Bulleted lists: LEFT-ALIGNED
- Numbered lists: LEFT-ALIGNED
- Footer text: CENTERED
- Legal disclaimer: CENTERED

## TEXT HIERARCHY (7 LEVELS MAX)

### Level 1: HEADER TITLE (H1 equivalent)
- Font size: 24px
- Line height: 1.2
- Weight: Bold (700)
- Alignment: Centered
- Color: White (on header) or ${text_primary} (in body)
- Usage: Main headline, maximum once per email

### Level 2: HEADER SUBTITLE
- Font size: 16px
- Line height: 1.3
- Weight: Normal (400)
- Alignment: Centered
- Color: #cbd5e1 (light gray)
- Usage: Optional subheadline under main title

### Level 3: BODY TEXT (Primary)
- Font size: 16px
- Line height: 1.5
- Weight: Normal (400)
- Alignment: Left-aligned
- Color: ${text_primary} (#1e293b or dark mode equivalent)
- Usage: Main content paragraphs

### Level 4: BODY TEXT (Secondary)
- Font size: 15px
- Line height: 1.5
- Weight: Normal (400)
- Alignment: Left-aligned
- Color: ${text_secondary} (#475569 or dark mode equivalent)
- Usage: Supporting text, less important information

### Level 5: FOOTER TEXT
- Font size: 12px
- Line height: 1.5
- Weight: Normal (400)
- Alignment: Centered
- Color: Various grays (#94a3b8, #cbd5e1, #e2e8f0)
- Usage: All footer content

### Level 6: CTA BUTTON TEXT
- Font size: 16px
- Line height: Normal (button height controls)
- Weight: Bold (700)
- Alignment: Centered
- Color: White (#ffffff)
- Usage: Primary call-to-action button only

### Level 7: UNSUBSCRIBE BUTTON TEXT
- Font size: 12px
- Line height: Normal
- Weight: Normal (400)
- Alignment: Centered
- Color: #94a3b8
- Usage: Unsubscribe link in footer

## FORBIDDEN TEXT STYLES
- NO italic text (except legal disclaimer in footer)
- NO underlines (except genuine links)
- NO text shadows, glows, or decorative effects
- NO letter-spacing adjustments (except 0-0.5px if necessary)
- NO text-transform: uppercase (except acronyms)
- NO multiple bold words within paragraphs (maximum 1-2 key terms)
- NO color gradients on text
- NO background colors on text (except highlights/callouts)

## TEXT COMPOSITION RULES
- Maximum 3 different font sizes per template
- Use the hierarchy strictly (don't skip levels)
- Paragraph spacing: 15-20px between paragraphs
- Line length: 50-75 characters per line optimal
- Prevent ugly hyphenation:
hyphens: none;
word-break: normal;
overflow-wrap: break-word;

- No manual hyphenation (no &shy;)

## MOBILE TYPOGRAPHY
- Body text: Minimum 16px for readability
- Headlines: 20-22px (slightly smaller than desktop)
- Line height: Slightly tighter (1.4 instead of 1.5)
- Button text: 16px minimum for tap targets

## COLOR & DARK MODE RULES (STRICT)

**NEVER** use hardcoded HEX colors (e.g., `#f4f6f9`, `#1e293b`) in inline styles for backgrounds or text.
**ALWAYS** use the following CSS variables:

### Mandatory Color Variables
- **Backgrounds:** `${content_bg}`, `${header_bg}`, `${card_bg}`, `${footer_bg}`
- **Text:** `${text_primary}`, `${text_secondary}`
- **Accents:** `${accent_color}`, `${border_color}`

### Mandatory CSS Classes for Dark Mode
Add these classes to ALL generated templates:
1.  `class="email-container dark-mode-base"` → Main 600px container `<table>`
2.  `class="dark-mode-text mobile-heading"` → All `<h1>`, `<h2>` headings
3.  `class="dark-mode-text mobile-text"` → All primary text `<p>` tags
4.  `class="dark-mode-text-secondary mobile-text"` → All secondary text `<p>` tags (where color is `${text_secondary}`)
5.  `class="dark-mode-header"` → Header `<td>` with `${header_bg}`
6.  `class="dark-mode-footer"` → Footer `<td>` with `${footer_bg}`
7.  `class="content-padding mobile-spacing"` → All `<td>` elements with padding
8.  `class="mobile-cta"` → Primary CTA `<a>` button

**Example of CORRECT vs. INCORRECT usage:**
<!-- ✅ CORRECT -->
<table style="background-color: ${content_bg};" class="dark-mode-base">
<h1 style="color: ${text_primary};" class="dark-mode-text mobile-heading">

<!-- ❌ INCORRECT -->
<table style="background-color: #f4f6f9;">
<h1 style="color: #1e293b;">
